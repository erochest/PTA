PTA Documentation

These scripts all take as input text files marked up with a form of SGML so restricted as to be almost worthless. You cannot use empty elements (with also specifying an end tag); there are no optional end tags; and there are only start and end tags and words and numbers. As I've said before, I'd wait for the next reincarnation.

There are five scripts included: text.pl, concord.pl, search.pl, freq.pl, coll.pl.

text.pl

Text.pl prints a list of all the words in the text, in the order in which they appear in the text, one word per line. This could be useful to give as input to other programs.

Usage: text.pl <file>

concord.pl

Concord.pl prints out a concordance of the text, citing each line by giving the line number in the file that the word occurs on and the character position of that word in the line.

Usage: concord.pl <file>

search.pl

Search.pl looks through the text for characters matching a Perl regular expression (see man perlre for more information on these). Notice that for some searches you may need to quote the regular expression so search.pl recognizes it as all one command line argument.

Usage: search.pl <file> <regular expression>

freq.pl

Freq.pl prints a list of all the words in the text and how many times they occur.

The "e;-f"e; command line option forces freq.pl to sort the results by frequency. The default sort order is alphabetic.

Usage: freq.pl <file> [-f]

coll.pl

Coll.pl prints all the collocates of all the words in a text.

The "e;-w"e; option specifies how many far to look to each side of each word. The default value is 5.

Usage: coll.pl <file> [-w <number>]
